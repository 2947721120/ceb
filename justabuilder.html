just a builder, natively scalable and designed for FRP


We would like a super button having the name super-button. 

var builder = ceb().name('super-button');

Our super-button must be an extension of the native button. So we will call it with <button is="super-button"></button> 

builder
    .extends('button')
    .prototype(Object.create(HTMLButtonElement.prototype));

The content of the button is linked to an attribute. But, this super button must be able to say hello to some one. So, we add the property who with a default value. 

builder.properties({
    label: {
        attribute: true,
        delegate: {
            target: 'span.label',
            property: 'textContent'
        }
    },
    who: {
        value: 'guy'
    }
});

To say hello, this super button has a super method sayHello. As you can see, the property who is used, but this is missing. Ouchhhh, it's possible? 

builder.methods({
    sayHello: function (el) {
        alert('Hello ' + el.who + '!');
    }
});

I forgot to explain the goal of this super button: say hello when it's clicked. 

builder.listen('click', function (el) {
    el.sayHello();
});

It's the scalable part of ceb. There, cebFeatureTemplate setup the content for our super button. It wraps the method createdCallback but it's another story ... 

builder.feature(cebFeatureTemplate, {
    template: '<span></span>'
});

The quick life of a builder start with a name and finish with register(). After its execution, the custom element is available. 

builder.register();

Ok, now we can play with it. 

<button is="super-button" label="click me!"></button>
