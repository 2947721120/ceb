We would like to share the behavior of a tooltip across elements. By this way we will be more DRY, but it’s not enough, we would like to hide the implementation of the tooltip too.

So, if for any reasons we switch for another UI library, we just have to change the tooltip implementation, not in each elements. In ceb, we can do that playing with features.

A feature is just a function.

    function tooltipFeature(el) {
        if (!el.__tooltipFeature) {
            el.__tooltipFeature = {};
        }
        return el.__tooltipFeature;
    }

With the previous function, we won’t do beautiful things in our case. It’s just the public API of the feature. The logic is done into the function tooltipFeature.setup.

    tooltipFeature.setup = function (struct, builder) {

The following functions will handle the life cycle of the tooltip. As you can see, the implementation is locked to bootstrap.

So, if we want to switch to another UI library, we just have to change these few lines.

        function apply(el, content) {
            if (!tooltipFeature(el).$tooltip) {
                tooltipFeature(el).$tooltip = $(el).tooltip({
                    title: content,
                    trigger: 'hover focus',
                    placement: 'auto bottom'
                });
            } else {
                var bsTooltip = tooltipFeature(el)
                    .$tooltip
                    .data('bs.tooltip');
                bsTooltip.options.title = content;
                bsTooltip
                    .tip()
                    .find('.tooltip-inner')
                    .text(content);
            }
        }

        function destroy(el) {
            if (tooltipFeature(el).$tooltip) {
                tooltipFeature(el).$tooltip.tooltip('destroy');
                tooltipFeature(el).$tooltip = null;
            }
        }

When the tooltip property is updated, the tooltip content must be updated too.

        builder.intercept('tooltip', function (
            next, el, propName, value
        ) {
            apply(el, value);
            return next(value);
        });

When the element is inserted into the DOM, the tooltip must be available.

        builder.wrap('attachedCallback', function (next, el) {
            next(arguments);
            apply(el, el.tooltip);
        });

When the element is removed from the DOM, the tooltip must be destroyed.

        builder.wrap('detachedCallback', function (next, el) {
            next(arguments);
            destroy(el);
        });

It’s done, the feature is ready to be used.

    };

Let’s go, we define a tooltiped input element.

    var builder = g.ceb().name('tooltiped-input').properties({
        tooltip: {
            attribute: true
        }
    }).feature(g.cebFeatureTemplate, {
        template: '<input type="text">'
    });

We add our freshly top mega fun tooltip feature to the structure.

    builder.feature(tooltipFeature);

We let the control to ceb and …

    builder.register();

… mouve your mouse over the input.

<tooltiped-input tooltip="I am a tooltip"></tooltiped-input>
