Feature: CQRS

This feature brings Command Query Responsibility Segregation to custom elements.

CQRS can be implemented from different ways. About custom elements, the implementation provided by ceb is based on the following flow:

  1. A command is sent.
  2. An handler process the command
  3. At the end the handler publishes an event
  4. Listeners process the event and update their state

Integration

ceb provides an implementation based on Rx. But, This feature is not locked to an external dependencies.

Send a command

To enable the sendCommand method, cebFeatureCqrs must be added to the structure.

ceb().name('cqrs-sender').feature(cebFeatureCqrs).build();

Create a new cqrs-sender.

var cqrsSend = document.createElement('cqrs-sender');
document.body.appendChild(cqrsSend);

Send the command using the sendCommand method.

cqrsSend.sendCommand('a-command', {
    id: 'id-1',
    name: 'name-1'
}, {
    user: 'current-user'
});

Publish an event

For enable the publishEvent method, cebFeatureCqrs must be added to the structure.

ceb().name('cqrs-publisher').feature(cebFeatureCqrs).build();

Create a new cqrs-sender.

var cqrsPublisher = document.createElement('cqrs-publisher');
document.body.appendChild(cqrsPublisher);

Send the command using the publishEvent method.

cqrsPublisher.publishEvent('an-event', {
 id: 'id-1',
 name: 'name-1'
}, {
 user: 'current-user'
});

Handle a command

The easiest way to create an handler is to used the handler builder: cebFeatureCqrs.handle.

var handler = cebFeatureCqrs

The builder start with the name of the event

  .handle('a-command')

prepare can be used to filter and work with the cmd object.

  .prepare(function (el, stream) {
      return stream.pluck('payload').filter(function (payload) {
          return payload.id === 'id-1'
      });
  })

process is used to do the business logic of the event.

  .process(function (el, cmd, payload) {
      console.log(el.tagName, payload)

The apply method will publish an event.

      cmd.apply('an-event', payload, {
          processedOn: new Date()
      })
  });

The built handlers must be given as options of the cebFeatureCqrs feature.

ceb().name('cqrs-handler').feature(cebFeatureCqrs, {
  handlers: [handler]
}).build();

Listen an event

The easiest way to create a listener is to used the listener builder: cebFeatureCqrs.listen.

var listener = cebFeatureCqrs

The builder start with the name of the event

  .listen('an-event')

prepare can be used to filter and work with the evt object.

  .prepare(function (el, stream) {
      return stream.pluck('payload').filter(function (payload) {
          return payload.id === 'id-1'
      });
  })

process is used to do the business logic of the event.

  .process(function (el, evt, payload) {
      console.log(el.tagName, payload)
  });

The built listeners must be given as options of the cebFeatureCqrs feature.

ceb().name('cqrs-listener').feature(cebFeatureCqrs, {
    listeners: [listener]
}).build();
